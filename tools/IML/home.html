Design Rationale
Core Principles
📶
Offline-First Architecture
Never blocks field officers due to connectivity. Recommendations are stored locally with encrypted retry queues. Data staleness is surfaced explicitly to the manager — not silently ignored.
🧠
Cognitive Load Reduction
Raw 3-month transaction data is pre-processed into AI-generated flag summaries. FOs see "what to pay attention to" — trend bars replace number tables. Anomalies surface automatically.
⚡
Manager Throughput Design
Batch processing restricted to Low-risk + FO-Approved + no-conflict cases only, preventing batch as a risk shortcut. Queue sorts by risk tier, amount, and conflict flag automatically.
♿
Accessibility as Constraint
Risk icons use distinct geometric shapes (▲ triangle / ◆ diamond / ● circle), not just color, satisfying WCAG 1.4.1 for deuteranopia and protanopia.
🔍
Transparency Architecture
FO reasoning note is mandatory (≥20 chars) and visible to the manager. Conflict cases show a data comparison — server state vs. FO review state. Managers must acknowledge conflicts.
🛡
No Silent Failure
Every error state shows a specific code, human-readable cause, safety assurance ("data saved"), and recovery action. Local data is never deleted without server receipt confirmation.

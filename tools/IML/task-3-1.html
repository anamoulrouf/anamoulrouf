SCREEN 01 — DASHBOARD & QUEUE
Design Annotations
01 · OFFLINE BANNER
Persistent Offline State
Orange pill in status bar + inline contextual banner. Never hidden — FO must always know data may be stale. Last sync timestamp displayed. Prevents uninformed decisions.
02 · RISK COUNTERS
Scannable Priority Summary
3-column counter (High/Med/Low) lets FO assess workload at a glance. Color-coded backgrounds reinforce urgency without text. Red count creates immediate visual hierarchy.
03 · LEFT BORDER + SHAPE
Dual Accessibility Signal
3px left border encodes risk tier for color sighted users. Shape icon (triangle/diamond/circle) provides the same signal for color-blind users. CONFLICT and PARTIAL DATA badges visible in list without opening.
04 · THUMB-ZONE DESIGN
One-Hand Operation
Primary tap targets (queue items, nav bar) in bottom 60% of screen. Top reserved for context/labels. Submit CTA always full-width at scroll bottom.
01 · AI SUMMARY BAND
Cognitive Load Reduction
Pre-computed bullet flags replace 3 months of raw data interpretation. FO sees "what to pay attention to" instead of raw figures. Color-coded dots encode severity without words.
02 · ELIGIBLE vs REQUESTED
Gap Visualization
Requested in red, eligible in green. Explicit "Gap" row shows the delta. FO immediately understands the overextension without mental arithmetic.
03 · PROGRESSIVE DISCLOSURE
Forced Linear Review
Profile → Transactions → Recommend is a forced linear flow. Prevents FOs from skipping the transaction review step and submitting uninformed recommendations.
01 · SPARKLINE BARS
Trend Over Numbers
Transaction data shown as relative bar heights, not absolute figures. FO's job is to spot trends. Month labels + trend arrows (↑↓) carry the signal. Tap "PARTIAL" button to demo the partial data state.
02 · PARTIAL DATA STATE
Interactive Toggle Demo
Tap "PARTIAL" in header to see missing months render as hatched grey bars. Coverage counter changes to amber "1/3". This is the exact rendering for Salma Akter's case in the queue.
03 · ANOMALY FLAG
System-Generated Alert
Unusual patterns auto-detected and surfaced as inline alerts. FO doesn't need to "notice" anomalies — system flags them. Reduces decision noise from missed signals.
01 · MANDATORY NOTE
Submit Gate
Submit button disabled until: (a) decision selected AND (b) note ≥20 chars. Forces structured reasoning, giving Manager transparency into "why." Character count shown live.
02 · CONTEXT REMINDER
Decision Support Band
Key AI flags repeated on Recommend screen. Prevents FO from making a decision based on memory of what they reviewed 2 screens ago.
03 · OFFLINE SAVE NOTICE
Trust Signal
Blue info banner explicitly states recommendation saved locally. FO doesn't worry about losing work. Reduces anxiety in field. Safety-first message, then action.
01 · FOUR STATES
All Sync States
Toggle Syncing / Success / Error / Empty states using the buttons above the content. Each has distinct visual treatment with clear next actions.
02 · ERROR STATE
No Data Loss Guarantee
"Data Safe: ✓ Saved locally" shown BEFORE the retry button. Sequencing matters — safety reassurance first, then action. Specific error code aids support escalation.
03 · SUCCESS STATE
Loop Closure
Server receipt confirmation shown — FO knows data reached Manager queue. Queue position gives context of Manager's workload. Pulsing ring provides satisfying closure.
